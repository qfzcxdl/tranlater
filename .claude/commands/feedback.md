必须使用以下mcp工具
### Interactive Feedback MCP（交互式反馈工具）

利用 `interactive-feedback-mcp` 工具，在任务完成时主动收集用户反馈，确保交付成果符合预期。

-**用途**：在每次会话结束或关键任务完成后，向用户请求反馈，以验证实现是否正确、是否需要调整。

-**使用时机**（必须调用）：
  1. **每次会话结束前**：无论任务大小，都应调用此工具收集用户反馈。
  2. **完成功能实现后**：代码编写完成，需要用户确认是否符合需求。
  3. **修复问题后**：Bug 修复完成，需要用户验证问题是否解决。
  4. **方案不确定时**：存在多种实现方案，需要用户选择或确认方向。

-**调用参数**：
  - `project_directory`: 项目的完整路径（必填）
  - `summary`: 简短描述本次变更或任务内容（必填，一行概述）

-**使用示例**：
  ```
  mcp_interactive-feedback-mcp_interactive_feedback(
    project_directory="/Users/xxx/code/project",
    summary="完成用户登录功能的 gRPC 接口实现"
  )
  ```

-**最佳实践**：
  - 在总结变更内容后立即调用，不要遗漏。
  - `summary` 应简洁明了，突出本次任务的核心内容。
  - 根据用户反馈决定是否需要进一步修改或优化。
  - 如果用户提出新需求，应在下一轮迭代中处理。

-**注意事项**：
  - 此工具是**强制要求**在每次会话中使用的。
  - 不要在任务中途频繁调用，应在阶段性成果完成后调用。
  - 确保 `project_directory` 使用绝对路径。
